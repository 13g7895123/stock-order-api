## 概述

儀表板目前所有數字皆由 `engine.py` 中的隨機模擬資料填入，  
若要上線實際交易，需將下列七大區塊的模擬資料替換為真實券商 API 回傳值。

資料來源分為兩種類型：

| 類型 | 說明 | 典型更新頻率 |
|------|------|-------------|
| **Push（推播）** | 券商主動推送，程式被動接收 | 即時（< 1 秒） |
| **Poll（輪詢）** | 程式主動定時查詢 | 5 秒 ～ 30 秒 |

## 一、統計卡列

儀表板頂部共 6 張統計卡，各卡所需資料如下。

| # | 卡片名稱 | 資料來源 | 取得方式 | 計算說明 |
|---|---------|---------|---------|---------|
| 1 | **今日損益** | 券商帳戶 API | Poll | 當日已實現損益 ＋ 所有持倉未實現損益加總（元） |
| 2 | **今日報酬率** | 自行計算 | — | 今日損益 ÷ 今日累積投入本金 × 100%，顯示至小數第二位 |
| 3 | **已實現損益** | 券商成交回報 | Push | 已賣出部位的損益，需扣除手續費（0.1425%）與交易稅（0.3%） |
| 4 | **持倉檔數** | 券商庫存查詢 | Poll | 目前帳戶中持有股票的不同代號數 |
| 5 | **今日交易檔數** | 券商成交回報 | Push | 當日買進成功的不同股票代號數 ／ 設定的每日上限（例：`3 / 5`） |
| 6 | **可用額度** | 券商帳戶 API | Poll | 帳戶可用資金餘額（元），即可立即下單的金額 |

> **注意**：今日損益與報酬率建議在每次成交回報推入後重新計算，而非僅依賴輪詢。

## 二、即時監控表格

儀表板中段左側，每列代表一檔監控中的股票。

| 欄位 | 資料類型 | 取得方式 | 更新頻率 | 說明 |
|------|---------|---------|---------|------|
| **代碼** | 基本資料 | 開盤前載入 | 靜態 | 股票代號，例：`2330` |
| **名稱** | 基本資料 | 開盤前載入 | 靜態 | 股票中文名稱，例：`台積電` |
| **價格** | 即時行情 | Push（Tick） | ≤ 1 秒 | 最新一筆成交價（元） |
| **漲跌** | 即時行情 | Push（Tick） | ≤ 1 秒 | 最新成交價 − 昨日收盤價（元），正值顯示綠色、負值顯示紅色 |
| **漲跌幅** | 自行計算 | — | ≤ 1 秒 | 漲跌 ÷ 昨日收盤價 × 100%，例：`+9.97%` |
| **委賣張數** | 即時五檔 | Push（五檔） | ≤ 1 秒 | **漲停價那一檔**的委賣張數（即第一筆委賣若價格等於漲停價時的掛單量） |
| **1 秒成交量** | 即時 Tick 累計 | Push（Tick） | ≤ 1 秒 | 滑動視窗：最近 1 秒內所有成交量的累加（張） |
| **起漲 K** | 策略內部 | 已有 | 策略觸發 | 目前偵測到的漲停 K 棒根數，策略自行計算，無需外部 API |
| **狀態** | 策略內部 | 已有 | 策略觸發 | `等待` / `準備進場` / `委託中` / `已進場` / `已完成` |
| **動作** | 無 | — | — | 「查看」按鈕，點擊可跳至該股票詳細頁面（未來功能） |

### 委賣張數判斷邏輯

```
取得五檔委賣後：
  if 委賣一檔價格 == 漲停價:
      委賣張數 = 委賣一檔數量
  else:
      委賣張數 = 0   # 代表漲停板已被打開或尚未漲停
```

## 五、成交記錄表格

儀表板底部右側，顯示當日所有已成交的買賣紀錄。

| 欄位 | 資料類型 | 取得方式 | 說明 |
|------|---------|---------|------|
| **時間** | 券商成交回報 | Push | 成交時間，格式 `HH:MM:SS` |
| **代碼** | 基本資料 | 開盤前載入 | 股票代號 |
| **名稱** | 基本資料 | 開盤前載入 | 股票中文名稱 |
| **類別** | 券商成交回報 | Push | `買進` / `賣出` |
| **價格** | 券商成交回報 | Push | 實際成交價（元） |
| **數量** | 券商成交回報 | Push | 實際成交張數 |
| **損益** | 自行計算 | — | 賣出時才計算並顯示：（賣出均價 − 買入均價）× 張數 × 1,000 − 手續費 − 交易稅 |

### 費用計算公式

```
手續費（買）= 成交金額 × 0.1425%，不足 20 元以 20 元計
手續費（賣）= 成交金額 × 0.1425%，不足 20 元以 20 元計
交易稅（賣）= 成交金額 × 0.3%（現股）
實際損益     = 賣出金額 − 買入金額 − 買入手續費 − 賣出手續費 − 交易稅
```

## 六、需串接的資料介面整理

### A. 即時行情（Tick 串流）

每筆成交時推送，需解析以下欄位：

```
{
  "code":       "2330",        // 股票代號
  "time":       "09:01:23",    // 成交時間
  "price":      1100.0,        // 成交價
  "volume":     50,            // 本筆成交量（張）
  "cum_volume": 1200,          // 當日累積成交量（張）
  "prev_close": 1000.0         // 昨日收盤價（用於計算漲跌）
}
```

### B. 即時五檔報價

每次五檔變動時推送，需解析以下欄位：

```
{
  "code":    "2330",
  "time":    "09:01:23",
  "ask": [                     // 委賣，ask[0] 為最低賣價（即漲停時的委賣一）
    {"price": 1100.0, "volume": 320},
    {"price": 1099.0, "volume": 150},
    ...
  ],
  "bid": [                     // 委買，bid[0] 為最高買價
    {"price": 1099.0, "volume": 88},
    ...
  ]
}
```

### C. 個股基本資料（開盤前一次性載入）

```
{
  "code":                   "2330",
  "name":                   "台積電",
  "market":                 "TSE",          // TSE（上市）/ OTC（上櫃）
  "prev_close":             1000.0,         // 昨日收盤價
  "limit_up_price":         1100.0,         // 漲停價（prev_close × 1.1，四捨五入至 tick）
  "limit_down_price":       900.0,          // 跌停價
  "prev_volume":            15000,          // 昨日成交量（張）
  "is_disposal":            false,          // 是否為處置股
  "is_attention":           false,          // 是否為注意股
  "is_day_trade_restricted": false,         // 是否為限當沖股
  "open_limit_up":          false           // 開盤即漲停（開盤後首筆 Tick 確認）
}
```

### D. 券商帳戶資訊（Poll，建議每 10 秒）

```
{
  "available_balance": 500000,    // 可用資金（元）
  "today_realized_pnl": 3200,     // 今日已實現損益（元）
  "positions": [
    {
      "code":       "2330",
      "qty":        2,             // 持有張數
      "avg_price":  1085.0         // 買入均價（元）
    }
  ]
}
```

### E. 券商委託回報（Push）

```
{
  "order_id":   "A123456789",
  "code":       "2330",
  "side":       "BUY",             // BUY / SELL
  "price":      1100.0,            // 0 代表市價
  "qty":        2,                 // 委託張數
  "filled_qty": 0,                 // 已成交張數
  "status":     "PENDING",         // PENDING / PARTIAL / FILLED / CANCELLED
  "time":       "09:01:22"
}
```

### F. 券商成交回報（Push）

```
{
  "order_id":   "A123456789",
  "code":       "2330",
  "name":       "台積電",
  "side":       "BUY",
  "price":      1100.0,
  "qty":        2,
  "time":       "09:01:23"
}
```

## 七、`engine.py` 待替換的模擬項目

| 目前模擬程式碼 | 替換為 |
|--------------|--------|
| `random.randint(10, 850)` 模擬 1 秒量 | Tick 串流接收後，以滑動視窗累計最近 1 秒成交量 |
| `random.random() > 0.75` 模擬漲停觸發 | 比對 Tick 成交價是否等於漲停價（`price == limit_up_price`） |
| `random.randint(5, 130)` 模擬委賣張數 | 五檔推播中，委賣一檔價格等於漲停價時取其張數，否則為 0 |
| `random.random() > 0.88` 模擬委買打開 | 五檔推播中，委買一檔價格等於漲停價時視為「打板成功，板子仍在」；若委賣一檔不再是漲停價則視為「板子打開」 |
| `MOCK_STOCKS` 固定清單 | 每日開盤前動態掃描全市場，篩選出「昨日非漲停、符合股價與成交量條件」的股票清單 |
| 統計卡全部顯示 `+0` / `0` | 接入帳戶 API 的損益、餘額、持倉數據 |

## 八、建議串接優先序

依照功能影響程度，建議按以下順序逐步替換：

```
優先序 1 ── 即時行情（Tick + 五檔）
            → 解鎖「即時監控表格」的價格、漲跌幅、委賣張數、1 秒量

優先序 2 ── 個股基本資料（開盤前載入）
            → 解鎖漲停價計算、特殊股篩選、市場分類

優先序 3 ── 券商成交回報（Push）
            → 解鎖「成交記錄」、「今日損益」、「已實現損益」

優先序 4 ── 券商委託回報（Push）
            → 解鎖「委託狀態表格」與撤單功能

優先序 5 ── 券商帳戶 API（Poll）
            → 解鎖「可用額度」、「持倉部位」精確庫存

優先序 6 ── 動態股票清單掃描
            → 替換 MOCK_STOCKS，實現每日自動選股
```
